Отвечаю на ваш вопрос и кратко описываю, как у нас устроена оплата.

1) Как мы создаём оплату (по API)
--------------------------------
Мы НЕ создаём ссылку «вручную» только в личном кабинете GetPlatinum для каждого платежа.

Схема такая:

• Пользователь на нашем сайте (sozdaipesnu.ru) выбирает пакет нот и нажимает «Оплатить».
• Наш backend вызывает API GetPlatinum:
  POST …/api/public/pay/init-payment-url
• В запросе передаём, в частности:
  - dealId — уникальный id заказа у нас (новый UUID на каждую попытку оплаты);
  - amount / positions (сумма, позиция, НДС);
  - clientParams:
      clientId  = постоянный id пользователя в нашей системе (один и тот же при повторных оплатах с одного аккаунта);
      email     = email аккаунта пользователя на нашем сайте (если есть);
      name      = пустая строка (ФИО с сайта не подставляем);
  - notificationUrl (webhook на наш сервер);
  - successUrl / failUrl (возврат на наш сайт).
• Телефон (phone) мы в API НЕ передаём — поля телефона у нас в init-payment-url нет.
• В ответе получаем formUrl и делаем редирект покупателя на эту ссылку GetPlatinum.

То есть оплата всегда создаётся через init-payment-url + редирект на formUrl.
Отдельной «своей» платёжной формы с телефоном на sozdaipesnu.ru у нас нет.


2) Ответ на ваш вопрос (про «сразу форма методов оплаты»)
--------------------------------------------------------
Вопрос:
«Верно понимаем, что у существующего покупателя вы инициализируете по API сразу
платёжную форму, где можно выбрать методы оплаты, минуя инициализацию заказа,
т.е. страницы, где покупатель может ввести свои данные?»

Ответ:
С нашей стороны вызов API всегда один и тот же: init-payment-url → formUrl.
Мы намеренно НЕ выбираем «режим без формы данных» отдельным флагом — такого
разветвления в нашем коде нет.

Наблюдение с нашей стороны (со стороны покупателя):

• Первый переход по formUrl: GetPlatinum показал страницу ввода данных
  (email / телефон / ФИО) — покупатель ввёл телефон с ошибкой.
• Повторные переходы с того же аккаунта нашего сайта (тот же clientId в clientParams):
  GetPlatinum сразу открывает экран выбора способа оплаты (QR / SberPay и т.д.),
  без повторного показа формы контактов. Редактировать телефон покупатель не может.

Мы предполагаем, что GetPlatinum «узнаёт» покупателя по clientId (и/или cookie /
профилю на стороне GP) и поэтому не показывает форму данных повторно.
Если это не так — подскажите, по какому ключу у вас склеивается «существующий
покупатель» при повторном init-payment-url с тем же clientId.

Итого: мы не «минуем» форму специально в API. Минуется она уже на стороне
GetPlatinum при повторном визите; formUrl каждый раз получаем заново через API.


3) В чём проблема у конкретного покупателя
------------------------------------------
• Покупатель (Максим, email: ________@________ — подставьте свой)
  при первой оплате указал неверный номер телефона на форме GetPlatinum.

  у этого покупателя в GetPlatinum ИЛИ подсказать, как при следующем
  init-payment-url заставить снова показать форму ввода данных
  (например, менять clientId на каждый заказ, отдельный параметр API, сброс профиля).
