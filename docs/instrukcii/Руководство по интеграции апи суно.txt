Руководство по интеграции API Suno
Прочитайте это перед интеграцией.
Данное руководство дополняет таблицы параметров на страницах API модели Suno. Страница API по-прежнему является источником достоверной информации о полях запроса, типах полей и примерах конечных точек, но в этом документе объясняются бизнес-правила, которые сложнее понять, опираясь только на схему.

Для получения точных определений полей и примеров запросов используйте вкладку API на странице модели Suno.
Используйте это руководство для выбора режима, правил комбинирования полей, обработки обратных вызовов и многоэтапных процессов.
Если вы интегрируете Suno в производственный процесс, обязательно прочтите это руководство перед написанием кода.
Что обычно сбивает команды с толку?
Создание песни — это не единый режим. На практике существует 4 распространенных комбинации запросов, и обязательные поля в каждой из них разные.
Функция Extend также имеет 2 режима. Будет ли результат воспроизводиться как вокальный или инструментальный, зависит от способа отправки prompt.
Процесс обработки целой песни может включать в себя несколько API. Во многих случаях сначала расширяется песня, и только потом запускается функция конкатенации.
Одного только вызова функции обратного вызова недостаточно, чтобы отметить задачу как успешно выполненную. Необходимо также проверить состояние на уровне задачи и содержимое результирующего файла.
Типичный рабочий процесс
Типичный процесс интеграции с Suno выглядит следующим образом:

Вызовите конечную точку создания задачи для нужной вам модели Suno.
Сохраните возвращенный результат taskId.
Дождитесь callBackUrlзавершения обработки веб-перехватчика или опросите конечную точку запроса задачи.
При стандартной генерации во многих случаях следует ожидать 2 варианта песни.
Если вам нужно продолжить воспроизведение уже записанной песни, запустите Extend.
Если после расширения вам нужна объединенная полноформатная песня, запустите функцию Concat на полученном расширенном результате.
1. 4 режима создания песен
В исходных документах описаны 4 варианта создания данных. Наиболее распространенная ошибка интеграции заключается в предположении, что некоторые поля можно просто опустить, хотя на самом деле их необходимо отправлять в виде пустых строк.

Режим	prompt	title	style	customMode/custom	instrumental	Примечания
Вокальная композиция из описания.	Необходимый	Пустая строка	Пустая строка	false	false	titleи styleдолжны присутствовать в виде пустых строк.
Инструментальная композиция из описания	Необходимый	Пустая строка	Пустая строка	false	true	Не пропускайте titleилиstyle
Песня с собственным вокалом	Необходимый	Необходимый	Необходимый	true	false	promptпредставляет содержание текста песни
Авторская инструментальная композиция	Пустая строка	Необходимый	Необходимый	true	true	promptследует отправлять как пустую строку, а не опускать.
1.1 Вокальная композиция из описания
Используйте этот режим, когда пользователь предоставляет описание нужной ему песни на естественном языке.

promptЭто описание песни.
titleдолжна быть пустой строкой.
styleдолжна быть пустой строкой.
customMode/ customдолжно быть false.
instrumentalдолжно быть false.
Пример:


Копия
{
  "input": {
    "prompt": "A happy and uplifting pop song",
    "title": "",
    "style": "",
    "customMode": false,
    "instrumental": false
  }
}
1.2 Инструментальная композиция из описания
Этот режим похож на предыдущий, за исключением того, что на выходе не должно быть вокала.

promptЭто по-прежнему описание, а не текст песни.
titleЭто должна быть пустая строка.
styleЭто должна быть пустая строка.
customMode/ customдолжно быть false.
instrumentalдолжно быть true.
Пример:


Копия
{
  "input": {
    "prompt": "A bright cinematic synthwave track for a night drive",
    "title": "",
    "style": "",
    "customMode": false,
    "instrumental": true
  }
}
1.3 Пользовательская вокальная песня
Используйте этот режим, если хотите напрямую управлять текстом песни, стилем и названием.

promptСодержит текст песни или черновик текста.
titleтребуется.
styleтребуется.
customMode/ customдолжно быть true.
instrumentalдолжно быть false.
На основе исходного материала:

promptне должно превышать 3000 символов.
styleне должно превышать 200 символов.
Пример:


Копия
{
  "input": {
    "prompt": "Write heartfelt lyrics about a summer romance by the sea",
    "title": "Summer Love",
    "style": "Warm pop, catchy, radio-friendly",
    "customMode": true,
    "instrumental": false
  }
}
1.4 Пользовательская инструментальная композиция
Это ещё один распространённый источник ошибок.

promptЭто должна быть пустая строка.
titleтребуется.
styleтребуется.
customMode/ customдолжно быть true.
instrumentalдолжно быть true.
Пример:


Копия
{
  "input": {
    "prompt": "",
    "title": "Night Drive",
    "style": "Synthwave, cinematic, neon atmosphere",
    "customMode": true,
    "instrumental": true
  }
}
2. Два режима расширения
Функция Extend используется, когда у вас уже есть сгенерированная песня Suno, и вы хотите продолжить её воспроизведение с определённой временной метки.

Наиболее важные области:

audio_id
continue_at
title
style
prompt
2.1 Расширение с помощью вокала
Если продолжение должно остаться вокальной песней:

promptдолжен содержать продолжение текста песни или вокальные указания.
titleдолжно иметь значение.
styleдолжно иметь значение.
continue_atуказывает системе, с какой секунды должно продолжаться воспроизведение оригинальной песни.
Пример:


Копия
{
  "input": {
    "audio_id": "song audio id",
    "style": "pop",
    "title": "Extended Song Title",
    "prompt": "Continue with a new chorus and a longer outro",
    "continue_at": 60
  }
}
2.2 Расширение инструментальной музыки
Если продолжение должно оставаться инструментальным:

promptЭто должна быть пустая строка.
titleдолжно иметь значение.
styleдолжно иметь значение.
Система различает инструментальное и вокальное продолжение в зависимости от того, promptпустое это пространство.
Пример:


Копия
{
  "input": {
    "audio_id": "song audio id",
    "style": "pop",
    "title": "Extended Song Title",
    "prompt": "",
    "continue_at": 60
  }
}
3. Для чего нужна функция Concat?
Функция Concat используется после Extension, когда вам нужна объединенная полная песня, а не только расширенный сегмент.

Основные моменты:

Передаваемый audio_idздесь параметр обычно является результатом audio_id, возвращаемым расширенным запросом.
Concat объединяет расширенную версию с оригинальным треком в цельную композицию.
Если вашему бизнесу нужен только сам сегмент расширения, этот шаг может и не потребоваться.
Пример:


Копия
{
  "input": {
    "audio_id": "8c9f6a88-672b-4b7c-93e0-31dbfdf42349"
  }
}
4. Как должна работать обработка запросов к задачам
Независимо от того, создаете ли вы, расширяете или объединяете элементы, вызов функции создания обычно сначала возвращает идентификатор задачи.

Таким образом, у вас есть два способа получить конечный результат:

Укажите callBackUrlвеб-перехватчик и дождитесь его получения.
Не указывайте URL-адрес обратного вызова и самостоятельно опрашивайте конечную точку запроса задачи.
Рекомендуемый подход:

Используйте callBackUrlдля рабочих процессов в бэкэнде.
Продолжайте использовать опрос в качестве резервного варианта для панелей мониторинга, отладки или задач сверки.
Избегайте создания производственного процесса, который зависит исключительно от опроса внешнего интерфейса.
5. Какие состояния обратного вызова следует проверять на самом деле?
Многие пользователи упускают из виду этот момент.

5.1 Состояние на уровне задач ApiPass
В текущих примерах использования обратных вызовов ApiPass ключевое состояние на уровне задачи обычно находится в следующем состоянии data.state:

success: задача успешно выполнена
fail: задача не выполнена
Типичный обратный вызов при успешном завершении:


Копия
{
  "code": 200,
  "data": {
    "taskId": "task_abc123xyz",
    "state": "success",
    "completeTime": 1755599644000,
    "costTime": 45000,
    "createTime": 1755599599000,
    "model": "suno/generate",
    "resultJson": {
      "data": [
        {
          "id": "audio_001",
          "audio_url": "https://cdn.apipass.dev/results/task_abc123xyz_0_audio.mp3",
          "image_url": "https://cdn.apipass.dev/results/task_abc123xyz_0_cover.jpeg",
          "duration": 180.5,
          "title": "Summer Love",
          "style": "Pop"
        }
      ]
    }
  },
  "msg": "Task completed successfully."
}
Типичный обработчик ошибки:


Копия
{
  "code": 501,
  "data": {
    "taskId": "task_abc123xyz",
    "state": "fail",
    "failCode": "GENERATION_FAILED",
    "failMsg": "Music generation failed due to content policy violation"
  },
  "msg": "Task failed."
}
В производстве безопасный минимум составляет:

Считайте задачу успешно выполненной только тогда, когда code === 200и data.state === "success".
Рассматривайте это data.state === "fail"как ветвь с критическим сбоем.
Настойчивость failCodeи failMsgустранение неполадок.
Не следует предполагать, что получение веб-хука автоматически означает успех.
5.2 Статус результатов Suno
В исходных документах каждый аудиообъект часто имеет собственное statusполе, например:

complete
Это значит, что вам следует различать:

data.stateдля завершения на уровне задачи
resultJson.data[].statusдля каждого возвращенного аудиофайла
Если ваш бизнес использует оба сгенерированных варианта, обрабатывайте элемент массива по отдельности, а не предполагайте, что существует только один результат.

5.3 callbackTypeВ исходных документах
В исходных документах также указано:

callbackType: "complete"
Это можно рассматривать как полезный дополнительный сигнал, но более безопасный подход всё же таков:

Проверятьdata.state
РазборresultJson.data
Перед переходом к следующему этапу работы необходимо проверить каждый результат.

7. Распространенные ошибки
7.1 Пропуск полей, которые должны быть пустыми строками
В некоторых режимах Suno отсутствие поля недопустимо. Требуется явно пустая строка. Наиболее распространенные случаи:

titleв режиме описания
styleв режиме описания
promptв пользовательском инструментальном режиме
7.2 Совмещение режима описания и пользовательского режима
Если у вас уже есть текст песни, название и стиль, используйте пользовательский режим. Если у вас есть только представление о песне на естественном языке, используйте режим описания.

prompt7.3 Обратное преобразование логики расширения
Расширение вокала: promptдолжно содержать контент
Инструментальное расширение: promptдолжна быть пустой строкой.
7.4 Отметка задачи как успешно выполненной сразу после получения веб-хука
Этого недостаточно. Как минимум, проверьте:

code
data.state
существует ли результирующий массив
проверяется, содержит ли каждый элемент результата идентификаторы и URL-адреса, необходимые для вашего рабочего процесса.
8. Рекомендуемый порядок выполнения
Если вы интегрируете Suno в реальный продукт, наиболее безопасная последовательность действий следующая:

Сначала разработайте процесс создания песни, а также обработку запросов и обратных вызовов.
Убедитесь, что обработчики событий успешного и неудачного выполнения сохраняются корректно.
Добавить расширение.
Добавляйте функцию Concat только после того, как предыдущие этапы стабилизируются.
Это гораздо безопаснее, чем пытаться запустить все ветки Suno одновременно.
