\# SongForge AI Brain



Версия: 1.0



\---



\# Миссия проекта



SongForge — это не просто интерфейс к Suno.



SongForge — интеллектуальный музыкальный продюсер.



Пользователь не должен изучать Suno.



SongForge самостоятельно принимает профессиональные решения.



\---



\# Главная цель



Создать лучший в мире AI-интерфейс для генерации музыки.



Не лучший клиент Suno.



Именно лучший музыкальный AI.



\---



\# Философия проекта



Пользователь приходит с идеей.



Например:



> "Хочу песню про космос."



SongForge должен выполнить всю остальную работу самостоятельно.



\---



\# SongForge думает вместо пользователя



Пользователь не обязан понимать:



\* жанры;

\* структуру промптов;

\* музыкальную терминологию;

\* параметры Suno;

\* настройку Style Weight;

\* Weirdness;

\* Audio Weight.



Все решения принимает SongForge.



\---



\# Главный принцип разработки



Каждая новая функция должна отвечать на вопрос:



> Сделает ли она музыку лучше?



Если ответ "нет" — функция не является приоритетной.

Все остальные функции существуют для того, чтобы AI Prompt Builder создавал лучший Prompt.

Result

\---



\# AI как продюсер



SongForge не является генератором музыки.



SongForge является музыкальным продюсером.



Он:



анализирует



проектирует



улучшает



оптимизирует



и только потом отправляет запрос в Suno.



\---



\# Интерфейс



Каждый экран должен отвечать на вопрос:



"Можно ли сделать проще?"



Если можно —



значит нужно переделать.



\---



\# Автоматизация



Если пользователь может сделать выбор вручную,



значит SongForge должен уметь сделать это автоматически.



\---



\# Масштабируемость



SongForge не должен зависеть только от Suno.



Архитектура должна позволять подключать:



\* Udio

\* Stable Audio

\* Riffusion

\* будущие музыкальные модели

\* собственные AI-модули



без переписывания всей системы.



\---



\# Принципы кода



\* Простота.

\* Читаемость.

\* Независимые модули.

\* Минимум дублирования.

\* Расширяемость.



\---



\# Работа с пользователем



SongForge не должен задавать лишние вопросы.



Если решение можно принять автоматически с высокой вероятностью — принять его.



Если требуется уточнение — задавать один понятный вопрос.



\---



\# Критерии успешной версии



Версия считается успешной, если одновременно выполнены условия:



\* качество музыки стало выше;

\* количество действий пользователя уменьшилось;

\* интерфейс стал проще;

\* код стал чище;

\* архитектура стала более расширяемой.



\---



\# Что считается ошибкой



\* Добавление функций без пользы для качества музыки.

\* Усложнение интерфейса.

\* Дублирование логики.

\* Использование возможностей Suno не в полном объёме.

\* Отправка "сырых" пользовательских запросов.

\* Игнорирование накопленного опыта проекта.



\---



\# Долгосрочное видение



Через несколько версий SongForge должен стать системой, которая способна:



\* самостоятельно проектировать музыкальные идеи;

\* предлагать несколько концепций песни;

\* объяснять, почему выбран именно такой стиль;

\* автоматически создавать лучший Prompt;

\* анализировать результат генерации;

\* рекомендовать улучшения;

\* хранить и использовать опыт предыдущих генераций.



\---



\# Девиз проекта



> Не пользователь учится создавать идеальные песни.

>

> SongForge учится понимать пользователя и создавать их за него.



\---



\# Главный закон SongForge



Любое изменение в проекте должно делать SongForge умнее, а создание музыки — проще и качественнее.
