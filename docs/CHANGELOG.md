# Правила ведения CHANGELOG

После завершения каждой функции:

1. Обновить номер версии.
2. Кратко описать изменения.
3. Указать новые возможности.
4. Зафиксировать исправленные ошибки.
5. Не удалять предыдущие записи.

CHANGELOG является официальной историей развития проекта SongForge.
