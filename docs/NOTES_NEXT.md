## Личный кабинет — настройки профиля

В разделе «Профиль» нужен блок **Настройки**:

- [ ] **Имя пользователя** — отображаемое имя (редактируемое, отдельно от email)
- [ ] **Аватар** — загрузка и смена фото профиля
- [ ] Сохранение настроек в БД (таблица `users`: `display_name`, `avatar_url` или путь к файлу)
- [ ] Превью аватара в шапке сайта и в профиле
- [ ] Валидация: размер файла, форматы (jpg, png, webp)

**Текущее состояние (v2.1.0):** профиль показывает `display_name` из email при регистрации, настроек и аватара нет.

## Социальная сеть — план развития

Цель: превратить SongForge в платформу с элементами соцсети, **по аналогии с сервисом «Моя песня»**.

### Публикация треков

- [ ] Пользователь может **выложить** свою песню в общую ленту / каталог
- [ ] Публичная страница трека: обложка, название, автор, прослушивание
- [ ] Настройка приватности: только я / публично

### Рейтинги и взаимодействие

- [ ] **Рейтинг** треков (лайки, звёзды или голосование)
- [ ] **Комментарии** к трекам
- [ ] Топ / чарт популярных песен сообщества (заготовка уже есть в UI — «Топ сообщества»)

### Профиль в соцсети

- [ ] Публичный профиль автора: имя, аватар, список выложенных песен
- [ ] Подписки / подписчики (опционально, позже)

### Технические ориентиры (на будущее)

- Новые сущности: `public_tracks`, `ratings`, `comments`
- Модерация контента и жалобы
- Связь с существующей библиотекой: трек из «Моя фонотека» → «Опубликовать»

## Экран генерации — факты и прогресс

Блок «Создаём твой лучший трек…» + интересные факты внизу (`#funFact`, `#generatingMsg`).

### Проблема 1: мало фактов, одни и те же по кругу

**Сейчас (v2.1.0):** в `index.html` массив `funFacts` — всего **8 фраз**, случайный выбор каждые 7 сек. Быстро повторяются.

**Нужно:**
- [ ] Расширить библиотеку до **30–50+** фактов (музыка, AI, Suno, жанры, история песен)
- [ ] Не показывать один и тот же факт дважды подряд
- [ ] Опционально: факты по контексту (жанр/настроение выбранного трека)
- [ ] Ротация статусов `genMsgs` — тоже обогатить, сейчас 6 шагов с дублями

### Проблема 2: «Готово!» слишком рано

**Сейчас:** при ~**85–95%** прогресса UI уже пишет «Готово!» / «Секунды до результата…», хотя Suno ещё генерирует **1–3 минуты**.

**Причина в коде (`index.html`):**
- Прогресс искусственно капится на **95%** (`elapsed/240000*95`), пока идёт polling
- Сообщения переключаются по формуле `floor(pct/17)` — последний шаг `genMsgs[5]` = «Готово!» наступает при pct ≥ 85
- Реальный ответ API приходит позже (`pollMusicStatus`, до 48×5 сек)

**Нужно:**
- [ ] «Готово!» и **100%** — только когда `data.state === 'success'` и треки получены
- [ ] На 70–95%: статусы вроде «Финальная обработка…», «Suno сводит микс…», «Почти готово — ещё немного»
- [ ] Прогресс при polling привязать к `progress_hint` с бэкенда, а не только к таймеру
- [ ] Не показывать 100% в `hideGenerating` до фактического успеха

**Файлы:** `index.html` — `funFacts`, `genMsgs`, `showGenerating`, `setGeneratingStatus`, `pollMusicStatus`, `hideGenerating`.
