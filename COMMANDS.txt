========================================
POLNAYA INSTRUKCIYA: OBNOVIT SERVER
========================================

SHAG 1. Otkryt PowerShell ili cmd na Windows

SHAG 2. Podklyuchitsya k serveru (skopirovat ODNU stroku):

ssh root@195.19.20.245

Vvesti parol servera.


SHAG 3. Obnovit sait (skopirovat ODNU stroku, vstaviti, Enter):

curl -fsSL "https://raw.githubusercontent.com/maxxx010182/SongForge/main/scripts/update-now.sh?$(date +%s)" | bash

Esli oshibka «staryj deploy-vps» ili TLS — SHAG 3b nizhe (NE update-now).


SHAG 3b. Obnovit sait VRUCHNUYU (esli SHAG 3 ne srabotal)
---------------------------------------------------------
Skopirovat PO ODNOMU (uzhe na servere posle ssh):

cd ~/SongForge

curl -fsSL "https://codeload.github.com/maxxx010182/SongForge/tar.gz/main" -o /tmp/sf.tar.gz

tar -xzf /tmp/sf.tar.gz -C /tmp SongForge-main/scripts/deploy-vps.sh

cp /tmp/SongForge-main/scripts/deploy-vps.sh scripts/deploy-vps.sh

bash scripts/deploy-vps.sh

Zhdat okonchaniya. V kontse dolzhno byt:

=== Gotovo! v2.11.13 ===

Deploy avtomaticheski zapuskaet:
  pm2: songforge        (API)
  pm2: songforge-worker (opros Suno + S3)


SHAG 4. Proverit versiyu (skopirovat ODNU stroku):

curl http://127.0.0.1:8000/api/health

Otvet: "version":"2.11.13"
Opcionalno: "redis":true, "s3":true, "email_smtp":true


SHAG 4b. SMTP — esli kod ne uhodit na pochtu

cd ~/SongForge
./venv/bin/python scripts/test_smtp.py

Esli FAIL — popravte .env, potom:

pm2 restart songforge

Podrobno: docs/instrukcii/SMTP-DIAGNOSTIKA.txt


SHAG 5. Redis (odin raz, esli net)

apt-get update && apt-get install -y redis-server
systemctl enable redis-server
systemctl start redis-server
redis-cli ping
Otvet: PONG


SHAG 6. S3 REG.RU v .env (odin raz)

nano ~/SongForge/.env

Podrobno: docs/instrukcii/S3-REG-RU-КАК-ЗАПОЛНИТЬ.txt

Posle .env:

pm2 restart songforge
pm2 restart songforge-worker


SHAG 7. (odin raz) Admin-panel — vash email v .env

ADMIN_BOOTSTRAP_EMAILS=vas@email.ru

Podrobno: docs/instrukcii/ADMIN-VHOD.txt

Esli menyali .env:

pm2 restart songforge
pm2 restart songforge-worker


SHAG 8. Vyiti s servera:

exit

KOMANDY PO OTDELNOSTI
========================================

=== PODKLYUCHIT K SERVERU ===
ssh root@195.19.20.245

=== OBNOVIT SAIT (obychno) ===
curl -fsSL "https://raw.githubusercontent.com/maxxx010182/SongForge/main/scripts/update-now.sh?$(date +%s)" | bash

=== OBNOVIT SAIT (esli TLS / raw.githubusercontent) ===
cd ~/SongForge
curl -fsSL "https://codeload.github.com/maxxx010182/SongForge/tar.gz/main" -o /tmp/sf.tar.gz
tar -xzf /tmp/sf.tar.gz -C /tmp SongForge-main/scripts/deploy-vps.sh
cp /tmp/SongForge-main/scripts/deploy-vps.sh scripts/deploy-vps.sh
bash scripts/deploy-vps.sh

=== OBNOVIT SAIT (esli i codeload ne kachaet — wget) ===
cd ~/SongForge
wget -O /tmp/sf.tar.gz "https://codeload.github.com/maxxx010182/SongForge/tar.gz/main"
tar -xzf /tmp/sf.tar.gz -C /tmp SongForge-main/scripts/deploy-vps.sh
cp /tmp/SongForge-main/scripts/deploy-vps.sh scripts/deploy-vps.sh
bash scripts/deploy-vps.sh

=== PROVERIT VERSIYU ===
curl http://127.0.0.1:8000/api/health

=== PM2: status API + worker ===
pm2 status
pm2 logs songforge-worker --lines 30 --nostream

=== ADMIN: redaktirovat .env ===
nano ~/SongForge/.env

=== ADMIN: perezapusk posle .env ===
pm2 restart songforge
pm2 restart songforge-worker

=== ADMIN: diagnostika (pochemu net prav) ===
cd ~/SongForge
./venv/bin/python scripts/admin_bootstrap_check.py
grep ADMIN_BOOTSTRAP .env

=== VYITI ===
exit

=== VOSSTANOVIT PROPAVSHUYU PESNYU (APIPass success, net v adminke) ===
cd ~/SongForge
./venv/bin/python scripts/check_task.py TASK_ID_IZ_APIPASS
./venv/bin/python scripts/recover_orphan_task.py TASK_ID --email EMAIL --prepaid --fix

=== V BRAUZERE ===
Ctrl+F5
